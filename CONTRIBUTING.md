## How to contribute to lib-anim

lib-anim is a volunteer effort and we encurage every contributions, from ideas and remarks to full code patches. There is no fixed roadmap for the project, but ideas for future developments are exposed in the [Issues](https://github.com/CandelierLab/lib-anim/issues) with the `enhancement` label. Anyone can contribute to this list.

#### **Do you have questions about the source code?**

* Ask any question about how to use lib-anim in the [Discussions](https://github.com/CandelierLab/lib-anim/discussions).

#### **Do you intend to add a new feature or change an existing one?**

* Suggest your change in the [Discussions](https://github.com/CandelierLab/lib-anim/discussions) or [Issues](https://github.com/CandelierLab/lib-anim/issues) with the `enhancement` label, and start writing code.

#### **Did you find a bug?**

* **Ensure the bug was not already reported** by searching on GitHub under [Issues](https://github.com/CandelierLab/lib-anim/issues).

* If you're unable to find an open issue addressing the problem, [open a new one](https://github.com/CandelierLab/lib-anim/issues/new). Be sure to include a **title and clear description**, as much relevant information as possible, and a **code sample** or an **executable test case** demonstrating the expected behavior that is not occurring.

#### **Did you write a patch that fixes a bug?**

* Open a new GitHub pull request with the patch.

* Ensure the PR description clearly describes the problem and solution. Include the relevant issue number if applicable.

#### **Did you fix whitespace, format code, or make a purely cosmetic patch?**

Changes that are cosmetic in nature and do not add anything substantial to the stability, functionality, or testability of lib-anim will generally not be accepted.

Thanks! :heart:
